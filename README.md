# Multi-AGV Path Planning Simulation (A* vs D*)

複数のAGV（無人搬送車）が衝突を回避しながら目的地へ移動するシミュレーションです。
静的な経路計画（A*）と、動的な経路再計画（D* like）の挙動の違いを比較検証できます。

[👉 デモをブラウザで試す](https://funmatu.github.io/multi-agv-path-planning-sim/)

## 概要

このシミュレーターでは、格子状のグリッドマップ上で複数のAGVを同時に制御します。
AGV同士の衝突や障害物を検知した際、以下の2つのアルゴリズムの違いによって全体の輸送効率がどう変わるかを可視化します。

### アルゴリズムの比較
1.  **A* (A-Star)**:
    * 最短経路を計算して移動します。
    * 進路上に他のAGVがいる場合、**「その場で待機」**して相手が通り過ぎるのを待ちます。
    * 単純ですが、デッドロック（膠着状態）が発生しやすい特徴があります。

2.  **D* (Dynamic A* / Replanning)**:
    * 障害や他のAGVを検知すると、即座に**「別のルートを再計算（Replan）」**して回避を試みます。
    * 計算コストはかかりますが、待機時間を減らし、動的な環境変化に強い特徴があります。

## シナリオ

シミュレーターには3つの検証用シナリオが用意されています。

* **Crossing (交差)**: 狭い交差点で複数のAGVがかち合う状況。譲り合いが発生します。
* **Bottleneck (ボトルネック)**: 中央の細い通路を双方向から通過する状況。効率的な合流が求められます。
* **Random**: ランダムに障害物とゴールを配置し、汎用的な性能をテストします。

## 機能・操作方法

* **Run A* / Run D***: シミュレーションを開始します。
* **Add Static Obstacle**: 動的に障害物を追加し、回避挙動を確認できます。
* **Simulation Speed**: AGVの移動速度を調整します（Fast/Normal/Slow）。
* **Number of AGVs**: 同時に稼働する台数を変更します（4〜10台）。
